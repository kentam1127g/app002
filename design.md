# 棒人間ゲーム 設計メモ

## コンセプト

GPT-4o-miniを使った棒人間の生活シミュレーター。
ユーザーが書いた日記や投稿した写真から棒人間が学習し、自分の言葉で喋ったり、
部屋のオブジェクトと自律的に関わったりする。

モチーフ：どこでもいっしょのトロ（単語を覚えて喋る）
独自性：パターン記憶による差分観察・ユーモアの自己生成・複数体の自律会話

リリース目標：Play Store（Phaser.js → Capacitor でラップ）

---

## 技術スタック

| レイヤー | 技術 |
|---|---|
| ゲームエンジン | Phaser.js |
| AI | OpenAI API（gpt-4o-mini） |
| Android化 | Capacitor（MVP後） |
| 状態保存 | localStorage（MVP）→ DB（後） |

APIキーはMVP段階では `config.js`（gitignore済み）で管理。
本番前にバックエンドプロキシへ移行する。

## メモリ構造

棒人間1体ごとに独立したメモリを持つ。

```
棒人間Aのメモリ
├── short_term[]     直近7日のイベントログ（期限切れで自動削除）
├── pattern{}        圧縮された傾向・習慣（永続保存）
├── word_bank[]      覚えた単語帳（永続保存）
├── quirks_bank[]    ユーモアのクセ（永続保存）
└── mood{}           patternから派生・都度更新
```

### short_term（直近7日）

生のイベントを保持する。

```json
{ "date": "2024-01-15", "type": "photo", "tags": ["自炊", "納豆", "朝食"] }
{ "date": "2024-01-14", "type": "diary", "words": ["疲れた", "残業"] }
```

### pattern（圧縮された傾向）

LLMが要約して生成するテキスト。「今日は納豆ないね」のような
差分観察を可能にする層。

```json
{
  "habits": "ほぼ毎日自炊。朝食に納豆が多い（週4〜5回）。",
  "mood_trend": "最近仕事が忙しそう。",
  "people": { "田中さん": { "count": 12, "vibe": "親しみがある" } }
}
```

### word_bank

棒人間が会話で使う素材。日記・写真から抽出した単語。
文字列配列ではなくオブジェクト配列で管理する。

```json
[
  {
    "word":     "田中さん",
    "weight":   7,
    "emotion":  "positive",
    "category": "person",
    "linked":   ["仕事", "疲れた"]
  },
  {
    "word":     "疲れた",
    "weight":   4,
    "emotion":  "negative",
    "category": "emotion",
    "linked":   ["田中さん", "仕事"]
  },
  {
    "word":     "ラーメン",
    "weight":   2,
    "emotion":  "neutral",
    "category": "food",
    "linked":   []
  }
]
```

**フィールド定義：**

| フィールド | 内容 |
|---|---|
| word | 単語本体 |
| weight | 登場回数。同じ単語が出るたびに+1 |
| emotion | positive / negative / neutral。APIが返す |
| category | person / food / place / emotion / event / other |
| linked | 同じ日記・写真から一緒に抽出された単語のリスト |

**linked の生成ルール：**

API側ではなくアプリ側で処理する。同一エントリから抽出された単語同士を自動的に相互リンクする。
同じペアが繰り返し共起するとそのリンクが強化され、会話のヒントとして使われる。

### quirks_bank

ユーモアのクセ。patternの更新タイミングで単語帳を元に生成。

```json
[
  { "type": "dajare",   "base": "ラーメン", "joke": "らめんね〜" },
  { "type": "rhyme",    "base": "疲れた",   "pair": "帰れた"    },
  { "type": "tenko",    "phrase": "そういうことか〜", "count": 5 },
  { "type": "favorite", "word": "ぴょん",   "reason": "響きが好き" }
]
```

### mood

patternから滲み出るもの。手動セットはしない。

```json
{ "label": "疲れ気味", "updatedAt": "2024-01-15" }
```

### patternの更新タイミング（C案採用）

棒人間が「読み返す」行動をとるタイミングで更新する。
ゲームイベントとして可視化できるため世界観に馴染む。
このタイミングでquirks_bankも生成・更新する。

### 容量目安

word_bankのオブジェクト化で1エントリあたりのサイズは増えるが、
単語数は緩やかにしか増えないため実用上は問題ない。

| 層 | 目安トークン |
|---|---|
| short_term（7日） | 〜700 |
| pattern | 〜200 |
| word_bank（50〜200語） | 〜600 |
| quirks_bank | 〜200 |
| **合計（1体）** | **〜1,700** |

3体で約5,100トークン。gpt-4o-miniのコンテキスト128kに対して余裕が25倍以上。

## 性格システム

### 基本方針（B案 + 初期テンプレート）

- 生成時にランダムでテンプレート性格を付与する
- 単語帳が蓄積するにつれて性格が変化していく
- 性格の文章をプロンプトに渡すだけでAIが行動を推論する

### 初期テンプレート例

マイペース・せっかち・几帳面・能天気・内向的・好奇心旺盛 など

### 性格の例（単語帳から派生）

```
word_bank: ["仕事", "疲れた", "締め切り"]
→ 性格テキスト：「真面目・几帳面・少し追い詰められやすい」

word_bank: ["コーヒー", "のんびり", "休日"]
→ 性格テキスト：「マイペース・くつろぎ重視・急がない」
```

## 発言スタイル（賢いポンコツ）

### 設計思想

**理解は完璧・表現は意図的に制限する。**

賢い人が語彙の少ない言語で話しているイメージ。
文脈の把握は深いが、出てくる言葉は少ない。

避けるべきパターン：「流暢だが的を外している」
→ LLMに「ポンコツっぽく」と指示すると理解を手加減しようとして起きる。

### プロンプト構造：思考と発言を分ける

```
[思考フェーズ]（外に出さない）
状況・文脈・パターンを完全に理解させる

[発言フェーズ]（出力）
理解した内容を制限された語彙で表現させる
```

### 発言ルール

- 15文字以内
- word_bankの単語を使う
- 単語一つだけのこともある
- 流暢な説明文は書かない
- 「…」「〜ね」「〜かも」を時々使う
- 的を外さない（理解した内容を最小限で表現する）

### 発言例

| 状況 | 発言 |
|---|---|
| 納豆のない写真 | 「…納豆」 |
| 田中さんが何度も出てくる | 「田中さん、また？」 |
| 久しぶりに明るい日記 | 「たのしそう！」 |
| よくわからないとき | 「…ふーん」 |

### 渡す単語の絞り込み

全単語を毎回渡すと文脈と無関係な単語をランダムに使う。
現在の文脈に関連性の高いもの（weight上位＋文脈近似語）に絞って渡す。

## 行動決定プロンプト（イメージ）

```
あなたは棒人間「{名前}」です。

性格：{性格テキスト}
気分：{mood}

[思考]
以下の情報から状況を完全に把握してください：
- 今起きていること：{イベント内容}
- 関連する記憶：{文脈に近いword_bankの抜粋}
- 最近の傾向：{pattern の抜粋}

[行動]
部屋にあるもの：
- 椅子（集中・作業・考える・一人になりたい）
- ソファ（くつろぐ・話したい・のんびり・深い話）
- ポスター（{写真から抽出した単語}）

場所を一つ選び、理由を10文字以内で。
形式：場所|理由
```

## ステート定義

AIロジックと視覚状態を管理する。8つで固定。

| ステート | 説明 |
|---|---|
| WALKING | 移動中 |
| IDLE | ぼーっとしてる・次の行動を考えてる |
| SITTING | 椅子などに座ってる |
| LOUNGING | ソファでくつろいでる |
| READING | 何かを読んでる（何の上かは問わない） |
| TALKING | 通常の会話（普通の距離・吹き出し） |
| WHISPERING | コソコソ話（密着・小さい吹き出し） |
| THINKING | 考え込んでる（頭上に...） |

---

## アニメーション分岐

ステートとアニメーションは分離する。

```
ステート      → AIロジック用（8つ固定）
onObject      → どのオブジェクト上にいるか
アニメーション → 描画システム用（自由に増やせる）
```

```
state=READING & onObject=chair → sit_read
state=READING & onObject=null  → stand_read
state=READING & onObject=sofa  → lounge_read
```

## オブジェクトスキーマ

```javascript
{
  id:       "sofa_01",
  type:     "sofa",
  keywords: ["くつろぐ", "眠い", "話したい", "深い話", "のんびり"],
  capacity: 2,           // 同時使用できる棒人間の数
  state:    "LOUNGING",  // このオブジェクト使用時のステート
  duration: [30, 90],    // 滞在秒数レンジ
  position: { x: 300 }
}
```

### オブジェクト一覧（案）

| type | keywords | capacity | state |
|---|---|---|---|
| chair | 集中・作業・考える・一人になりたい | 1 | SITTING |
| sofa | くつろぐ・話したい・のんびり・深い話 | 2 | LOUNGING |
| poster | 写真から抽出した単語（動的） | 1 | THINKING |
| window | 外・ぼんやり・遠くを見る・物思い | 1 | IDLE |
| mailbox | 日記・手紙・届いた | 1 | READING |

## 会話システム

### 同じ場所にいる時の会話トリガー

座った瞬間ではなく「しばらく隣にいたら」発火させる。

```
B がソファに着席
    ↓
2〜4秒、お互い無言でLOUNGING
    ↓
近接判定が一定時間続いたら TALKING へ移行
```

すれ違い（BがソファへAが立ち上がる）はバグではなくゲームの可愛げとして許容する。

### 会話の種類

- **通常会話**：TALKING（普通の距離・吹き出し）
- **コソコソ話**：WHISPERING（密着・小さい吹き出し）
- **単語の伝播**：一方が知らない固有名詞を「知ってる？」と聞くことで別の棒人間に伝わる

### 会話フォーマット（LLM出力）

```json
{ "type": "whisper", "from": "A", "to": "B", "text": "田中さんって誰だろ", "pause_after": true }
```

## プライバシー

- 日記はOpenAI APIに送信される（30日間保持される）
- 固有名詞は除外せず、プライバシーポリシーに明記して開示する
- 学んだ単語をユーザーが一覧・削除できるUIを設ける
- 日記入力時に「AIによる処理が行われます」の表示を添える

## 棒人間の数

- 初期：1体
- 最大：3〜4体（様子を見ながら増やす）

## ゲームのテンポ

半放置ゲームのリズムを基本とする。
頻繁に動かさない。ユーザーがたまに覗いたとき「あ、変わってる」と気づく温度感。

> 日記を書いてアプリを閉じる → 数時間後に開くと棒人間の様子が変わっている

## 行動決定のタイミング

| フェーズ | 方法 |
|---|---|
| MVP | 3〜5分ごとに行動決定（放置ゲームのリズム） |
| 次 | イベント駆動を追加（日記到着・会話終了など） |
| その後 | LLMがnext_action_inを返して自律制御 |

**コスト試算（放置ゲーム版）：**
3体 × 4分ごと × 1時間 = 45回/時間。慌ただしい版の約1/8。

## 抽出プロンプト仕様

### 単語抽出の設計思想

固有名詞（人名・地名）はゲームの「個人性」の核心であり、
ユーザーの生活が部屋に滲み出る部分。
一度見逃すとそのエピソードが永久に失われるため、別枠で必ず抽出する。

感情語・食べ物・出来事などは翌日以降の日記でも出てくるため、
日記の長さに応じた上限を設ける。

### 抽出ルール

```
枠1（固有名詞）：人名・地名を全て抽出、最大3個まで
枠2（その他）  ：日記の長さに応じてN個

合計が多くなっても固有名詞は省かない
```

| 文字数 | 枠2（その他）の上限 |
|---|---|
| 〜100文字 | 2個 |
| 100〜200文字 | 3個 |
| 200〜300文字 | 4個 |

### 抽出プロンプト

```
以下の優先順位でこの日記から単語を抽出し、JSON配列で返してください。

優先順位：
1. 人名・地名（category: person または place）を全て含める（最大3個）
2. 感情語を1〜2個（category: emotion）
3. その他の印象的な単語をN個（food / event / other）

フィールド：word / emotion（positive・negative・neutral）
           / category（person・food・place・emotion・event・other）

例：[{"word":"田中さん","emotion":"positive","category":"person"},
    {"word":"疲れた","emotion":"negative","category":"emotion"}]

説明文は不要。JSON配列のみ返す。
```

### 抽出例

```
日記：「田中さんと山田さんと渋谷に行ってラーメンを食べた。疲れたけど楽しかった。」

→ [
    {"word":"田中さん", "emotion":"positive", "category":"person"},
    {"word":"山田さん", "emotion":"neutral",  "category":"person"},
    {"word":"渋谷",    "emotion":"neutral",  "category":"place"},
    {"word":"疲れた",  "emotion":"negative", "category":"emotion"},
    {"word":"ラーメン", "emotion":"positive", "category":"food"}
  ]
```

### 写真からの単語抽出

gpt-4o-miniのビジョン機能を使用。画像をbase64で渡す。

```javascript
{
  role: 'user',
  content: [
    { type: 'image_url', image_url: { url: `data:image/jpeg;base64,${base64}` } },
    { type: 'text', text: 'この写真から印象的な単語を3〜5個。JSON配列で。フィールド：word / emotion / category' }
  ]
}
```

写真は日記投稿時に任意で添付できる（Bプラン採用）。
